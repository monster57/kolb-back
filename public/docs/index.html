provide the same that have given in password field.

GET  /api/user/startup

this api will get startup regiistration data related to the current user. This api will work different for different users. if user type is startup it will give the startup data related to the user. If the user is relationship manager it will give all the registration data which are not rejected and not aproved. If the user is senior manager the api will give only the startup data which are aproved by the relationship manager. If the user is nodal officer then it will take all the startup data which are aproved by the senior manager. Last if the user is managing director then the user will give only startup which are aproved by the nodal officer

Body

Response

Parameter Type Description
Success Boolean

This will give the success of the request that have been send to the Api
message String

The messege that will give the understanding of the result
data Object

this will contain the registration data of the startup

POST  /api/startup/register

this api will create or update startup registration form

Query Params

Body

Parameter Type Description
basic_details Object

this will contain the basic details of the startup.

This has key fields.

name <String> this will contain the name of the startup

url <String> this field contains the url of the startup website

1. `name` <String> This will contain the name of the image
2. `contentType` <String> This will contain the type of the image
3. `data` <String> this will hold the image data
